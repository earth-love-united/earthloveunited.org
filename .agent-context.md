> This file is the quick-reference for common operations.

## Before You Edit

1. **Read `ARCHITECTURE.md`** for module map, z-index stack, and event flows
2. All `const IIFE` modules **MUST** have `window.MODULE = MODULE;` after the IIFE
3. The `safeCall()` / `hasModule()` / `safeGet()` functions use `window[name]` — NOT `typeof` checks
4. **NEVER** append interactive DOM elements as children of `#globeViz` (z-index: 1 trap)
5. CSS `pointer-events` cascade: parent `none` → children `none` (unless child explicitly sets `auto`)
6. Elements hidden with `opacity: 0` or `transform: translateX(100%)` still capture clicks unless `pointer-events: none` is also set
7. The `.sections` container has `background: var(--bg)` — it's opaque and captures events above the globe

## Module Wiring Pattern

```javascript
// ✅ CORRECT — accessible via safeCall/hasModule
const MY_MODULE = (() => {
  // ... private implementation ...
  return { publicMethod };
})();
window.MY_MODULE = MY_MODULE;  // ← THIS LINE IS REQUIRED

// ❌ WRONG — invisible to safeCall
const MY_MODULE = (() => {
  return { publicMethod };
})();
// Missing window.MY_MODULE = MY_MODULE → safeCall silently fails
```

## Before You Ship

1. Run the boot validator: open console, look for 🔴 errors from `[BOOT]`
2. Check no `position:fixed` element has `pointer-events:auto` when it should be hidden
3. Click a globe point → verify GLOBE_OVERLAY left sidebar opens
4. Scroll to quiz → verify quiz buttons are clickable
5. Check console for `[safeCall] X.Y() threw:` warnings

## Common Traps

| What | Why | Fix |
|------|-----|-----|
| `safeCall('X', 'method')` returns `undefined` | `window.X` not set | Add `window.X = X;` after IIFE |
| Overlay/panel invisible | Appended to `#globeViz` (z:1) | Append to `document.body` |
| Invisible element blocking clicks | `opacity:0` but `pointer-events:auto` | Add `pointer-events:none` |
| Off-screen panel stealing clicks | `transform:translateX(100%)` without `pe:none` | Add `pointer-events:none` to default state |
| `#globe-overlay` behind content | Inside `#globeViz` stacking context | Must be child of `body`, not `#globeViz` |

## Script Load Order

Scripts in `index.html` load synchronously. A module can only reference modules above it.
`gaia-utils.js` (safeCall) loads first. `app.js` (init) loads last.
`module-validator.js` loads just before `app.js` and validates all modules are on `window`.

## Dev Tools (Console Commands)

These tools are loaded from `tools/` and available in the browser console:

| Command | What It Does |
|---------|-------------|
| `SmokeTest.run()` | Run all 22 tests across 5 categories (modules, stacking, dom, interactions, resources) |
| `SmokeTest.run('modules')` | Run only module binding tests |
| `SmokeTest.table()` | Show last results as a table |
| `StackLint.audit()` | Full z-index audit: fixed elements, invisible blockers, stacking traps, drift detection |
| `StackLint.quick()` | Silent check — only logs if issues found |
| `StackLint.watch()` | Re-run quick check every 3s (useful during development) |
| `Tracer.start()` | Start logging all `safeCall`/`hasModule` calls in real-time |
| `Tracer.stop()` | Stop tracing |
| `Tracer.report()` | Show call frequency table and failure analysis |
| `Tracer.graph()` | Show caller → callee dependency graph |
| `Tracer.failed()` | Show only failed cross-module calls |

### When to Use Each Tool

- **After any CSS change**: `StackLint.audit()` — catches stacking context regressions
- **After adding/editing a module**: `SmokeTest.run('modules')` — catches missing `window.X`
- **When something "silently doesn't work"**: `Tracer.start()` → interact → `Tracer.failed()` — catches silent `safeCall` failures
- **Before shipping**: `SmokeTest.run()` — full validation
